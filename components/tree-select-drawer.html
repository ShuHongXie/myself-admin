MlTreeSelectDrawer 树形选择抽屉 ​

树形选择抽屉组件，将树形选择功能集成到抽屉中，支持单选、多选等功能。

基础用法 ​

点击按钮打开抽屉，在抽屉中选择树形数据。

选择部门

多选抽屉 ​

通过 multiple 属性支持多选功能。

选择权限（多选）

事件透传示例 ​

所有 el-tree 的事件都会自动透传，可以直接在父组件中监听。

事件日志：

事件透传示例

事件透传原理

组件使用 v-on="$attrs" 实现事件批量透传，无需手动定义每个事件。

透传链路：

父组件 → MlTreeSelectDrawer → MlTreeSelect → el-tree

这意味着你可以直接在 <ml-tree-select-drawer> 上监听所有 el-tree 的事件，如：

  • @node-click - 节点被点击
  • @node-contextmenu - 节点右键菜单
  • @check-change - 勾选状态变化
  • @check - 勾选节点
  • @current-change - 当前选中节点变化
  • @node-expand - 节点展开
  • @node-collapse - 节点收起
  • @node-drag-start - 节点开始拖拽
  • @node-drag-enter - 拖拽进入节点
  • @node-drag-leave - 拖拽离开节点
  • @node-drag-over - 拖拽经过节点
  • @node-drag-end - 拖拽结束
  • @node-drop - 拖拽成功

抽屉特有属性 ​

参数说明类型可选值默认值

继承 MlTreeSelect 的所有属性，如 treeData、multiple、nodeKey 等。

组件自定义事件 ​

事件名说明回调参数
confirm确认选择时触发(selectedData: T | T[]) => void
close抽屉关闭时触发() => void

el-tree 透传事件 ​

所有 el-tree 的事件都会自动透传，包括：

事件名说明回调参数
node-click节点被点击时触发(data, node, instance)
node-contextmenu节点被鼠标右键点击时触发(event, data, node, instance)
check-change节点选中状态发生变化时触发(data, checked, indeterminate)
check点击复选框时触发(data, checkState)
current-change当前选中节点变化时触发(data, node)
node-expand节点被展开时触发(data, node, instance)
node-collapse节点被关闭时触发(data, node, instance)
node-drag-start节点开始拖拽时触发(node, event)
node-drag-enter拖拽进入其他节点时触发(draggingNode, dropNode, event)
node-drag-leave拖拽离开某个节点时触发(draggingNode, dropNode, event)
node-drag-over在拖拽节点时触发(draggingNode, dropNode, event)
node-drag-end拖拽结束时触发(draggingNode, dropNode, dropType, event)
node-drop拖拽成功完成时触发(draggingNode, dropNode, dropType, event)

使用场景 ​

  • 在页面侧边显示树形选择器
  • 移动端或需要大面积展示的树形选择
  • 组织架构选择、菜单选择等
  • 相比对话框更适合复杂的树形结构展示

注意事项 ​

  1. 抽屉关闭时重置：关闭抽屉时会自动重置树形选择的状态
  2. 属性透传：抽屉的其他属性（如 close-on-click-modal、lock-scroll 等）可以直接传递
  3. 事件透传：所有 el-tree 的事件都会自动透传，无需手动定义
  4. 双向绑定：务必使用 v-model 绑定 visible 属性，否则抽屉无法正确打开/关闭
  5. 方向选择：通过 direction 属性可以控制抽屉从哪个方向打开：
    • rtl：从右向左（默认）
    • ltr：从左向右
    • ttb：从上到下
    • btt：从下到上
